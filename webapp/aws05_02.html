AWS Storage Gateway, 파일 게이트웨이용 Amazon S3 Intelligent-Tiering 추가

Data : Apr 30, 2020

AWS Storage Gateway는 이제 Amazon S3 Intelligent-Tiering을 파일 게이트웨이용 스토리지 클래스로 지원하여 데이터 액세스 패턴 변동 시 성능에 영향을 주지 않거나 운영 오버헤드를 초래하지 않으면서 스토리지 비용을 자동으로 최적화합니다. 이를 통해 온프레미스 사용자 및 애플리케이션 데이터 액세스 패턴에 대해 깊이 이해하지 않아도 됩니다.

파일 게이트웨이는 SMB 및 NFS 프로토콜을 사용하여 온프레미스 애플리케이션에서 파일 기반의 캐싱 방식으로 사실상 무제한의 클라우드 스토리지에 액세스할 수 있도록 합니다. 이 출시로 인해 이제 파일 게이트웨이에 파일 공유를 설정할 때 Amazon S3 Intelligent-Tiering을 스토리지 클래스로 선택할 수 있습니다.

파일 게이트웨이를 Amazon S3 Intelligent-Tiering에 데이터를 저장하도록 구성하면 액세스 패턴이 변경될 때 자주 액세스하는 티어와 자주 액세스하지 않는 티어 간에 데이터를 이동하여 자동 비용 절감 효과를 누릴 수 있습니다. 자주 변동되는 액세스 패턴이 있는 워크로드 또는 변동적인 액세스 패턴이 있는 여러 애플리케이션에서 액세스하는 데이터가 있는 경우 이상적인 솔루션입니다.

이 기능은 오늘부터 AWS Storage Gateway를 사용할 수 있는 모든 리전에서 새로운 게이트웨이에 대해 사용할 수 있습니다. 기존 게이트웨이에 대해서는 2020년 5월 30일까지 소프트웨어 업데이트를 통해 사용할 수 있습니다. 단계별 지침은 AWS Storage Gateway 사용 설명서를 참조하십시오. 스토리지 클래스 옵션과 요금 정보는 Amazon S3 페이지에서 확인하십시오.
