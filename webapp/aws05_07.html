Amazon EFS 파일 시스템에 대한 Amazon ECS 및 AWS Fargate 지원 기능 정식 출시

Data : Apr 8, 2020

Amazon Elastic Compute Cloud(EC2) 및 AWS Fargate에서 모두 실행되는 Amazon Elastic Container Service(ECS) 태스크는 이제 Amazon Elastic File System(EFS) 파일 시스템을 마운트할 수 있습니다.

EFS를 사용하는 ECS 태스크는 태스크 정의에서 고객이 지정한 파일 시스템을 자동으로 마운트하고 리전의 모든 가용 영역에서 태스크의 컨테이너에 이를 사용할 수 있도록 합니다. 이렇게 하면 ECS의 태스크 및 컨테이너 수준에서 일관성 있는 공유 스토리지를 정의 및 사용할 수 있습니다.

Wordpress, Drupal 및 JIRA 같은 애플리케이션은 EFS를 사용하여 가용성을 높이기 위해 다른 가용 영역에서 실행 중인 활성/대기 컨테이너 쌍으로 사용자 데이터 및 애플리케이션 상태를 저장합니다. 데이터 과학 팀은 EFS를 사용하여 공유 교육 데이터 및 홈 디렉터리를 저장하므로 노트북 컨테이너 및 분산 교육 작업이 동시에 데이터에 액세스할 수 있습니다.

ECS와 함께 EFS를 사용하기 위해 고객은 ECS 작업 정의에 하나 이상의 볼륨 정의를 추가할 수 있습니다. 볼륨 정의에는 EFS 파일 시스템 ID, 액세스 포인트 ID, 그리고 전송 중 IAM 권한 부여 또는 TLS 암호화의 활성화 여부가 포함됩니다. 작업 정의 내의 컨테이너 정의는 컨테이너가 실행될 때 마운트되어야 할 작업 정의 볼륨을 지정할 수 있습니다. EFS 파일 시스템을 사용하는 작업이 실행되면 ECS는 EFS 파일 시스템이 마운트되어 이에 액세스해야 하는 컨테이너가 사용할 수 있도록 해 줍니다.

Amazon ECS의 EFS 지원은 ECS 및 EFS가 지원되는 모든 리전에서 사용 가능합니다. 시작하려면 설명서를 참조하십시오.
